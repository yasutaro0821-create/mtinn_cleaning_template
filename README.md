# mt. inn 清掃管理システム v4.0

Google スプレッドシート + Apps Script + **PWA** で清掃管理を自動化するプロジェクトです。
ハウスキーピングスタッフがiPhoneから快適に操作できるPWAと、フロントが使うスプレッドシートを双方向で同期します。

## 解決する課題

| 課題 | 解決策 |
|------|--------|
| iPhoneでスプレッドシートのプルダウンが小さすぎる | PWAの大きなボタンでステータス変更 |
| フロント↔ハウスキーピングの情報共有が遅い | 30秒間隔の自動同期で双方向リアルタイム共有 |
| 誰が清掃/点検したか不明確 | スタッフ選択式ログインと担当者必須入力 |
| 近眼のスタッフが画面を読めない | 大フォント(16px〜24px)・大ボタン(48px以上)のUI |

## システム構成

```
iPhone (PWA)  ◀──▶  GAS Web App (API)  ◀──▶  Google Spreadsheet
  清掃スタッフ         doGet / doPost          データストア (唯一の真実)
                                                     ▲
PC (ブラウザ)  ───── スプレッドシート直接操作 ──────────┘
  フロントスタッフ
```

## 機能

### 既存機能（スプレッドシート + GAS）
- ねっぱん CSV から当日予約を自動取り込み
- 泊数・3日清掃・チェックアウトを自動判定し色分け
- 清掃/点検担当の入力を必須化し、ログを蓄積
- 40室の部屋割りボードを自動生成

### 新機能（PWA）
- **スタッフ選択ログイン**: 名前をタップするだけ（パスワード不要）
- **部屋一覧画面**: 40室をフロア別グリッド表示、ステータス色分け
- **ワンタップステータス変更**: 大きな色付きボタンで操作
- **責任明確化**: 清掃完了→清掃担当必須、清掃点検済→点検担当必須
- **双方向同期**: PWA⇔スプレッドシート 30秒間隔自動同期
- **進捗サマリー**: 残り部屋数・完了率をプログレスバーで表示
- **PWA対応**: ホーム画面に追加可能、アプリのように動作

## 清掃指示シート生成ロジック

### ねっぱんCSV取り込み
1. Google Drive の `neppan_csv` フォルダから当日日付(YYYYMMDD)を含むCSVを検索
2. Shift_JIS エンコードで読み込み、柔軟なヘッダーマッチングで列を検出
3. X列（24列目）の `n | d` 形式（例: `2 | 3` = 3泊中2泊目）を解析

### 清掃判定ルール

```javascript
isCheckin       = (n === 1)                    // チェックイン日
isStay          = (n >= 2)                     // 連泊（2泊目以降）
isThirdCleaning = (isStay && n % 3 === 0)      // 3泊ごとのフル清掃（3,6,9泊目...）
isCheckoutToday = (d > 0 && n === d)           // チェックアウト日
isFullCleaning  = isThirdCleaning || isCheckoutToday  // フル清掃が必要
```

### 泊数別の清掃要否と色分け

| 泊目 | 行の背景色 (A,C-G列) | B列(部屋番号) | 清掃 |
|------|----------------------|---------------|------|
| 1泊目 (チェックイン) | `#CCFFCC` 黄緑 | — | — |
| 2泊目 | なし | — | 不要 |
| 3泊目 | `#CCFFFF` 水色 | `#FFCCCC` ピンク | フル清掃 |
| 4泊目 | なし | — | 不要 |
| 5泊目 | なし | — | 不要 |
| 6泊目 | `#CCFFFF` 水色 | `#FFCCCC` ピンク | フル清掃 |
| チェックアウト日 | — | `#FFCCCC` ピンク | フル清掃 |

### 清掃指示シートの列構成（A〜Q列）

| 列 | 内容 | 備考 |
|----|------|------|
| A | 泊数 (例: 1/3) | neppan X列から変換 |
| B | 部屋番号 | 3桁 (301-420) |
| C | 氏名 | ゲスト名 |
| D | プラン名 | 商品プラン |
| E | 人数 | 合計人数 |
| F | 大人 | 大人人数 |
| G | 子供 | 子供人数 |
| H-M | ロジックフラグ | 非表示（isCheckin, isStay, isThirdCleaning, isCheckoutToday, isFullCleaning, isAmenityOnly） |
| N | ステータス | プルダウン（6種） |
| O | 清掃担当 | スタッフプルダウン |
| P | 点検担当 | スタッフプルダウン |
| Q | 更新日時 | MM/dd HH:mm |

## ステータスと色

| ステータス | 色 | 意味 |
|-----------|-----|------|
| 在室 | `#CCFFCC` 緑 | ゲストが部屋にいる |
| チェックアウト | `#DDDDDD` グレー | ゲスト退出済み・清掃待ち |
| 連泊鍵預け中 | `#CCE5FF` 青 | 連泊中・鍵を預けて外出 |
| 清掃中 | `#FFF2CC` 黄 | 清掃作業中 |
| 清掃完了 | `#FFD699` オレンジ | 清掃完了・点検待ち |
| 清掃点検済 | `#99E699` 濃緑 | 点検完了・販売可能 |

## 対象ユーザーと使い方

### フロントスタッフ（PC）
1. 毎朝ねっぱんCSVをDriveに保存
2. スプレッドシートで「清掃管理 → 清掃指示シート生成」を実行
3. 各部屋のステータスを設定（在室/チェックアウト/連泊鍵預け中）

### ハウスキーピングスタッフ（iPhone PWA）
1. PWAを開いて名前を選択
2. 部屋一覧で清掃対象を確認（色で一目瞭然）
3. 部屋をタップ → 「清掃中」ボタン → 清掃実施
4. 完了後 → 名前を確認 → 「清掃完了」ボタン

### 点検者（iPhone PWA）
1. 「清掃完了」（オレンジ）の部屋を確認
2. 部屋をタップ → 点検実施 → 「清掃点検済」ボタン

## 入出力

### 入力
- **自動取得**: ねっぱん CSV（部屋番号/氏名/プラン/人数/泊数/CI/CO）
- **フロント入力**: スプレッドシートでステータス設定
- **ハウスキーピング入力**: PWAでステータス変更・担当者選択

### 出力（スプレッドシート — 唯一の真実のソース）
- 清掃指示シート（A〜Q列）
- 部屋割りボード（40室グリッド）
- 清掃ログ（日時・部屋・ステータス・担当者）

## API エンドポイント

| メソッド | パラメータ | 説明 |
|---------|-----------|------|
| GET | `?action=rooms` | 全部屋のステータス一覧を取得 |
| GET | `?action=staff` | スタッフ名一覧を取得 |
| GET | `?action=summary` | 進捗サマリーを取得 |
| POST | `action=updateStatus` | ステータスを更新 |

## 開発・デプロイ

### 前提条件
- Node.js + npm
- clasp（`npm install -g @google/clasp`）
- Google アカウントで clasp login 済み

### 開発コマンド
```powershell
cd C:\Users\yasut\mtinn_cleaning_template
clasp status   # 変更確認
clasp push     # ローカル → GAS
clasp pull     # GAS → ローカル
```

### PWAデプロイ
1. `clasp push` でGASにアップロード
2. GASエディタ → デプロイ → ウェブアプリとして公開
3. 「次のユーザーとして実行: 自分」「アクセスできるユーザー: 全員」
4. 発行されたURLをiPhoneのホーム画面に追加

## スプレッドシートのシート構成

| シート名 | 用途 | 生成方法 |
|---------|------|---------|
| 清掃指示 | 当日の清掃指示データ（A〜Q列） | メニュー「清掃指示シート生成」で自動生成 |
| 部屋割り | 40室のステータスボード（10×4グリッド） | 清掃指示生成時・ステータス変更時に自動更新 |
| 清掃ログ | 清掃完了・点検済のログ（日時・部屋・担当） | ステータスが「清掃完了」「清掃点検済」時に自動追記 |
| スタッフ一覧 | 清掃担当/点検担当の名前リスト | メニュー「スタッフ初期設定」で初期化 |
