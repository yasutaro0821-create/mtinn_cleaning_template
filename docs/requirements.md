# mt. inn 清掃管理PWA 要件定義書

## 1. プロジェクト概要

### 1.1 背景
mt. inn（40室のホテル）では、ねっぱん（ホテル管理システム）のCSVデータをGoogle スプレッドシートに取り込み、清掃管理表を運用している。しかし、ハウスキーピングスタッフがiPhoneでスプレッドシートを操作する際、プルダウンが小さすぎてステータス変更ができない問題がある。

### 1.2 目的
- iPhoneでも快適に操作できるPWA（Progressive Web App）を構築
- スプレッドシートとの双方向リアルタイム同期を維持
- フロントとハウスキーピング間の双方向コミュニケーションを実現
- 清掃・点検の責任者を明確化しミスを削減
- 近眼のスタッフでも見やすいUI/UXを提供

### 1.3 対象ユーザー
| ユーザー | デバイス | 主な操作 |
|---------|---------|---------|
| フロントスタッフ | PC（スプレッドシート） | 朝の清掃指示シート生成、ステータス確認・変更 |
| ハウスキーピングスタッフ | iPhone（PWA） | ステータス変更、担当者選択 |
| 管理者 | PC/iPhone | 全体進捗確認、設定変更 |

## 2. システム構成

### 2.1 アーキテクチャ
```
┌─────────────────┐     ┌──────────────────────┐     ┌──────────────────┐
│  iPhone (PWA)   │────▶│  GAS Web App (API)   │────▶│ Google Sheets    │
│  清掃スタッフ    │◀────│  doGet / doPost      │◀────│ (データストア)    │
└─────────────────┘     └──────────────────────┘     └──────────────────┘
        ▲                                                     ▲
        │                                                     │
        └─────── 双方向同期（ポーリング30秒間隔）──────────────┘
                                                              │
┌─────────────────┐                                           │
│  PC (ブラウザ)   │───── スプレッドシート直接操作 ─────────────┘
│  フロントスタッフ │
└─────────────────┘
```

### 2.2 技術スタック
| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | HTML5 + CSS3 + Vanilla JS | 追加ライブラリ不要、GASから配信可能 |
| バックエンドAPI | Google Apps Script (doGet/doPost) | 既存GASを拡張、追加サーバー不要 |
| データストア | Google Spreadsheet | 既存資産を活用、フロントとの双方向性を維持 |
| ホスティング | GAS Web App | 無料、Google認証連携可能 |
| PWA | Service Worker + manifest.json | オフライン対応、ホーム画面追加 |

### 2.3 既存GASとの関係
- 既存の `main.gs` はそのまま維持（朝のシート生成、onEdit等）
- PWA用のAPI関数を追加（`doGet`を拡張、`doPost`を新設）
- スプレッドシートが「唯一の真実のソース」として機能し続ける

## 3. 清掃指示シート生成ロジック（main.gs）

### 3.0.1 ねっぱんCSV取り込みプロセス
1. Google Drive `neppan_csv` フォルダから当日日付(YYYYMMDD)を含むCSVファイルを検索
2. Shift_JIS エンコードで読み込み、`Utilities.parseCsv()` でパース
3. ヘッダー行から柔軟にカラムを検出（キーワードマッチング）:
   - 部屋: `部屋`, `部屋名`, `客室`, `ルーム`, `Room`, `部屋番号`
   - 氏名: `利用者氏名`, `氏名`, `お名前`, `名前`, `宿泊者`, `代表者`
   - プラン: `商品プラン`, `プラン`, `商品名`, `プラン名`, `Plan`
   - 泊数: X列(24列目) の `n | d` 形式 → `n/d` に変換
4. 各行に対し清掃判定ロジックを適用
5. 清掃指示シートに書き込み＆背景色設定
6. 部屋割りボードを自動更新

### 3.0.2 清掃判定ルール
```
n = 現在の泊数（何泊目か）
d = 合計泊数

isCheckin       = (n === 1)                    // チェックイン日
isStay          = (n >= 2)                     // 連泊（2泊目以降）
isThirdCleaning = (isStay && n % 3 === 0)      // 3泊ごとのフル清掃
isCheckoutToday = (d > 0 && n === d)           // チェックアウト日
isFullCleaning  = isThirdCleaning || isCheckoutToday
```

### 3.0.3 背景色ルール

**行の背景色（A列, C-G列）:**
| 条件 | 色 | 意味 |
|------|-----|------|
| `isCheckin` (n=1) | `#CCFFCC` 黄緑 | 本日チェックイン |
| `isThirdCleaning` (n%3===0) | `#CCFFFF` 水色 | 連泊清掃日（3,6,9泊目） |
| それ以外の連泊 | なし | 清掃不要 |

**B列（部屋番号）の背景色:**
| 条件 | 色 | 意味 |
|------|-----|------|
| `isFullCleaning` | `#FFCCCC` ピンク | フル清掃必須 |

**泊数ごとの早見表:**
| 泊目 | 行の色 | B列 | 清掃 |
|------|--------|------|------|
| 1泊目 | 黄緑 | — | — |
| 2泊目 | なし | — | 不要 |
| 3泊目 | 水色 | ピンク | フル清掃 |
| 4泊目 | なし | — | 不要 |
| 5泊目 | なし | — | 不要 |
| 6泊目 | 水色 | ピンク | フル清掃 |
| CO日 | — | ピンク | フル清掃 |

### 3.0.4 清掃指示シート列構成（A〜Q列）
| 列 | 内容 | ソース |
|----|------|--------|
| A | 泊数 (例: 1/3) | neppan X列 |
| B | 部屋番号 | neppan 部屋列 |
| C | 氏名 | neppan 氏名列 |
| D | プラン名 | neppan プラン列 |
| E | 人数 | neppan 人数列 |
| F | 大人 | neppan 大人列 |
| G | 子供 | neppan 子供列 |
| H | isCheckin | 自動算出（非表示） |
| I | isStay | 自動算出（非表示） |
| J | isThirdCleaning | 自動算出（非表示） |
| K | isCheckoutToday | 自動算出（非表示） |
| L | isFullCleaning | 自動算出（非表示） |
| M | isAmenityOnly | 自動算出（非表示、将来用） |
| N | ステータス | プルダウン（6種類） |
| O | 清掃担当 | スタッフプルダウン |
| P | 点検担当 | スタッフプルダウン |
| Q | 更新日時 | 自動記録（MM/dd HH:mm） |

## 4. 機能要件（PWA）

### 4.1 画面一覧

#### 画面1: ログイン画面（スタッフ選択）
- スタッフ一覧シートから名前リストを取得
- **大きなボタン**で名前を選択（タップしやすいサイズ 48px以上）
- 選択したスタッフ名はlocalStorageに保存（次回起動時に自動ログイン）
- ログインすると自分の名前がヘッダーに常時表示

#### 画面2: 部屋一覧画面（メイン画面）
- 40室をフロア別にグリッド表示（3F: 301-320、4F: 401-420）
- 各部屋カードに表示する情報:
  - 部屋番号（大きく太字、24px以上）
  - 現在のステータス（色付き背景）
  - ゲスト名（存在する場合）
  - 泊数（例: 1/3）
  - 清掃担当・点検担当（アイコンまたは名前）
- ステータスに応じた色分け（既存色を踏襲）:
  - 在室: 🟢 `#CCFFCC`
  - チェックアウト: ⚪ `#DDDDDD`
  - 連泊鍵預け中: 🔵 `#CCE5FF`
  - 清掃中: 🟡 `#FFF2CC`
  - 清掃完了: 🟠 `#FFD699`
  - 清掃点検済: ✅ `#99E699`
- フロアフィルター（3F / 4F / 全て）
- ステータスフィルター（未清掃のみ / 清掃待ちのみ / 全て）
- **自動更新**: 30秒ごとにデータ再取得（他ユーザーの変更を反映）
- プルダウンリフレッシュ対応

#### 画面3: 部屋詳細・ステータス変更画面
- 部屋カードをタップすると遷移（またはモーダル表示）
- 表示情報:
  - 部屋番号（大きく）
  - ゲスト名、プラン名
  - 泊数、人数（大人/子供）
  - チェックイン/チェックアウトフラグ
  - フル清掃かどうか
  - 現在のステータスと更新日時
  - 清掃担当・点検担当
- ステータス変更: **大きなボタン**（各ステータスに対応）
  - 次の有効なステータスのみ表示（例: 「在室」→「チェックアウト」のみ）
  - ボタンサイズ: 高さ60px以上、幅100%
  - 色付きボタン（ステータス色と一致）
- 担当者選択:
  - 清掃担当（清掃完了に変更時に必須）
  - 点検担当（清掃点検済に変更時に必須）
  - ログインスタッフ名を自動入力候補として提示
- 変更確認ダイアログ（誤タップ防止）
- 変更完了後、スプレッドシートに即時反映

#### 画面4: 進捗サマリー画面
- 全体の清掃進捗を一目で確認
  - 未着手 / 清掃中 / 清掃完了 / 清掃点検済 の部屋数
  - プログレスバー（視覚的に進捗把握）
- 本日の清掃ログ一覧（時系列）

### 4.2 バリデーション（既存ルール踏襲）
- 「清掃完了」→ 清掃担当(O列)の入力が必須
- 「清掃点検済」→ 点検担当(P列)の入力が必須
- 未入力の場合はエラーメッセージ表示（ステータスは変更されない）

### 4.3 双方向同期
- **PWA → スプレッドシート**: ステータス変更時に即時書き込み（doPost API）
- **スプレッドシート → PWA**: 30秒間隔のポーリングで最新データ取得
- フロントがPCでスプレッドシートを直接編集した場合も、PWA側に30秒以内に反映
- 楽観的UI更新: API呼び出し前にUIを即時更新し、失敗時にロールバック

### 4.4 オフライン対応（PWA）
- Service Workerでアプリシェルをキャッシュ
- オフライン時はデータ取得不可の旨を表示
- ステータス変更はオンライン復帰後に送信（キューイング）

### 4.5 通知機能（将来拡張）
- フロントがステータスを変更した際にPWAへ通知（Phase 2）

## 5. 非機能要件

### 5.1 UI/UXデザイン原則
| 項目 | 要件 |
|------|------|
| フォントサイズ | 最小16px、部屋番号24px以上、ステータス20px以上 |
| タッチターゲット | 最小48×48px（Apple HIG準拠） |
| 色分け | 既存6色ステータスを踏襲、コントラスト比4.5:1以上 |
| レスポンシブ | iPhone SE〜iPhone 15 Pro Max対応 |
| 操作ステップ | ステータス変更は最大3タップ（部屋選択→ステータス選択→確認） |

### 5.2 パフォーマンス
- 初期表示: 3秒以内
- ステータス変更のAPI応答: 2秒以内
- ポーリング間隔: 30秒（GAS実行時間制約を考慮）

### 5.3 セキュリティ
- GAS Web Appの実行権限: 「自分として実行」「全員がアクセス可」
  - または「ドメイン内のユーザーのみ」（Workspace利用の場合）
- スタッフ選択式ログイン（パスワード不要、運用簡素化のため）

### 5.4 ブラウザ対応
- Safari (iOS 15+) — 必須
- Chrome (Android) — 対応
- Chrome / Edge (PC) — 対応

```json
      "roomNo": "301",
      "guest": "山田太郎",
      "plan": "1泊2食",
      "pax": "2",
      "adults": "2",
      "children": "0",
      "isCheckin": true,
      "isStay": false,
      "isThirdCleaning": false,

      "cleaner": "",
      "checker": "",
      "updatedAt": ""
    }
```

スタッフ一覧取得

**レスポンス例:**
```json
{

  "checkers": ["高橋", "山本"]
```

ステータス更新

**リクエスト:**
```json
{
  "action": "updateStatus",
  "row": 2,
  "roomNo": "301",
  "newStatus": "清掃完了",
  "cleaner": "田中",
  "checker": ""
}
```

**レスポンス:**
```json
{
  "status": "ok",
  "message": "301号室のステータスを「清掃完了」に更新しました",
  "updatedAt": "03/01 09:15"
}
```

進捗サマリー取得

## 10. 運用フロー（PWA導入後）

### 朝のフロー
1. **フロント（PC）**: ねっぱんCSVをDriveに保存
2. **フロント（PC）**: スプレッドシートで「清掃指示シート生成」を実行
3. スプレッドシートに40室のデータが展開される
4. フロントが各部屋のステータスを設定（在室/チェックアウト/連泊鍵預け中）

### 日中フロー
5. **ハウスキーピング（iPhone PWA）**: アプリを開く
6. 部屋一覧で今日のステータスを確認（色付き）
7. 清掃する部屋をタップ → ステータスを「清掃中」に変更
8. 清掃完了 → 自分の名前を選択 → 「清掃完了」ボタン
9. **点検者（iPhone PWA）**: 「清掃完了」の部屋を確認
10. 点検完了 → 自分の名前を選択 → 「清掃点検済」ボタン

### リアルタイム同期
- ハウスキーピングがPWAで変更 → スプレッドシートに即時反映
- フロントがスプレッドシートで変更 → PWAに30秒以内に反映
- 全スタッフが常に最新の状態を共有
